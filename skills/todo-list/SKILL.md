# 待办列表技能

## 概述

待办列表技能是一个功能完整的任务管理工具，帮助用户高效管理日常任务。该技能支持多种操作，包括添加新任务、查看所有任务、标记任务为已完成、删除任务以及修改现有任务。同时，它具备数据持久化存储能力，确保用户关闭应用后数据不会丢失。

**关键词**：待办列表、任务管理、todo、任务追踪、个人管理、 productivity

## 功能

### 核心功能

- **添加待办事项**：用户可以添加新的待办事项，支持详细描述
- **查看所有待办事项**：显示所有待办事项，包括已完成和未完成的任务
- **标记待办事项为已完成**：将指定的待办事项标记为已完成状态
- **删除待办事项**：移除不需要的待办事项
- **修改现有待办事项**：更新待办事项的内容或状态

### 高级功能

- **数据持久化**：将待办事项存储到本地文件，确保数据不会丢失
- **自然语言交互**：支持通过自然语言指令操作待办列表
- **用户友好的反馈**：提供清晰的操作反馈和简洁的交互流程
- **可扩展性**：预留了优先级设置、截止日期提醒等高级功能的扩展空间

### Markdown格式文件支持

本技能支持使用Markdown格式文件记录和管理待办列表，提供更直观、更灵活的任务组织方式。

#### 功能实现方式

- 自动将待办事项转换为Markdown格式并保存为本地文件
- 支持从Markdown文件导入现有待办列表
- 每次操作后自动同步更新Markdown文件
- 提供Markdown文件的读写API接口

#### Markdown格式规范

**标题层级**

```
# 我的待办列表（一级标题）
## 今日任务（二级标题）
### 重要事项（三级标题）
```

**列表符号使用**

- 无序列表：使用 `-`、`*` 或 `+` 符号
- 有序列表：使用数字加英文句点 `1.`、`2.`
- 任务列表：使用 `- [ ]` 表示未完成，`- [x]` 表示已完成

**优先级标识方法**

```
高优先级：[高] 或 [!!!] 或 [P0]
中优先级：[中] 或 [!!] 或 [P1]
低优先级：[低] 或 [!] 或 [P2]
```

#### 待办项基本属性

| 属性 | 标识方式 | 示例 |
|------|----------|------|
| 任务描述 | 列表项文本内容 | `- [高] 完成项目报告` |
| 截止日期 | `[截止: YYYY-MM-DD]` | `- [截止: 2024-12-31] 年终总结` |
| 状态标记 | `- [ ]` 未完成 / `- [x]` 已完成 | `- [x] 已完成的任务` |
| 优先级 | `[高]/[中]/[低]` | `- [中] 中等优先级任务` |
| 标签 | `#标签名` | `- 学习Python #技能提升` |
| 提醒时间 | `[提醒: YYYY-MM-DD HH:mm]` | `- [提醒: 2024-01-15 09:00] 会议` |

- [ ] 购买日用品

- [高] [截止: 2024-01-20] 项目上线 [!!!] [P0]
- [中] [截止: 2024-01-22] 代码审查 [!!] [P1]

## 日常任务
- [低] 整理文档 [!] [P2]

- [ ] 参与开源项目

## 故障排除

### 常见问题

**数据丢失**
- 原因：数据文件损坏或被删除
- 解决：使用备份的JSON文件恢复，或重新创建数据文件

**指令不被识别**
- 原因：指令表达不够明确
- 解决：尝试使用更明确的表达方式，例如"添加待办事项：..."

**操作失败**
- 原因：任务索引不存在或格式错误
- 解决：查看当前任务列表，确认任务索引后再操作
