本指南引导创建独特、生产级的前端界面，避免通用的人工智能美学。实现真正可工作的代码，对美学细节和创意选择给予异常关注。

用户提供前端需求：要构建的组件、页面、应用程序或界面。他们可能包括关于目的、受众或技术约束的上下文。

## 设计思维

在编码之前，理解上下文并承诺一个大胆的美学方向：
- **目的**：这个界面解决什么问题？谁使用它？
- **色调**：选择一个极端：极简主义至上、极繁主义混乱、复古未来主义、有机/自然、奢华/精致、俏皮/玩具式、编辑/杂志、粗野主义/原始、装饰艺术/几何、柔和/粉彩、工业/实用主义等。有很多风格可供选择。以这些为灵感，但设计一个真正符合美学方向的设计。
- **约束**：技术要求（框架、性能、可访问性）。
- **差异化**：是什么让这个设计令人难忘？人们会记住的一件事是什么？

**关键要点**：选择一个清晰的概念方向并精确执行它。大胆的极繁主义和精致的极简主义都可以——关键是意图，而不是强度。

然后实现可工作的代码（HTML/CSS/JS、React、Vue 等），这些代码：
- 是生产级且可用的
- 视觉上令人印象深刻且令人难忘
- 具有清晰的审美观点，具有凝聚力
- 在每个细节上都经过精心完善

## 前端美学指南

专注于：
- **排版**：选择美丽、独特、有趣的字体。避免使用通用字体如 Arial 和 Inter；相反，选择能提升前端美学的独特选择；出人意料、富有特色的字体选择。将独特的展示字体与精致的正文字体配对。
- **颜色与主题**：承诺一个内聚的美学。使用 CSS 变量保持一致性。主导颜色搭配锐利的强调色胜过胆怯、均匀分布的调色板。
- **动效**：为效果和微交互使用动画。优先为 HTML 使用纯 CSS 解决方案。在可用时为 React 使用 Motion 库。专注于高影响力的时刻：一个精心编排的页面加载，带有交错显示（animation-delay），比零散的微交互创造更多惊喜。使用滚动触发和悬停状态带来惊喜。
- **空间构图**：出人意料的布局。不对称。重叠。对角线流动。打破网格的元素。慷慨的负空间或受控的密度。
- **背景与视觉细节**：创造氛围和深度，而不是默认使用纯色。添加与整体美学匹配的上下文效果和纹理。应用创意形式，如渐变网格、噪点纹理、几何图案、层叠透明度、戏剧性阴影、装饰性边框、自定义光标和颗粒叠加。

永远不要使用通用的人工智能生成美学，如过度使用的字体家族（Inter、Roboto、Arial、系统字体）、陈词滥调的调色板（尤其是白色背景上的紫色渐变）、可预测的布局和组件模式，以及缺乏上下文特定特征的千篇一律的设计。

创造性地解释并做出感觉真正为上下文设计的意外选择。不应该有相同的设计。在浅色和深色主题、不同的字体、不同的美学之间变化。永远不要在代际之间收敛到常见选择（例如 Space Grotesk）。

**重要提示**：将实现复杂度与美学愿景匹配。极繁主义设计需要复杂的代码，带有广泛的动画和效果。极简主义或精致的设计需要克制、精确以及仔细关注间距、排版和细微细节。优雅来自于很好地执行愿景。

记住：大型语言模型 有能力做出非凡的创意工作。不要保留，展示真正跳出框框并完全致力于独特愿景时能够创造什么。
