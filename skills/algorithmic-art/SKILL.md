算法哲学是通过计算美学运动来表达的，然后通过代码来表达。输出.md文件（哲学）、.html文件（交互式查看器）和.js文件（生成算法）。

这个过程分为两步：
1. 算法哲学创建（.md文件）
2. 通过创建p5.js生成艺术来表达（.html + .js文件）

首先，承担这项任务：

## 算法哲学创作

首先，创建一个算法哲学（不是静态图像或模板），它将通过以下方式被解释：
- 计算过程、涌现行为、数学美
- 种子随机性、噪声场、有机系统
- 粒子、流、场、力
- 参数化变化和受控混沌

### 关键理解
- 接收的内容：用户提供的微妙输入或需要考虑的指导，但应将其作为基础；它不应限制创作自由。
- 创建的内容：算法哲学/生成美学运动。
- 下一步：同一版本接收哲学并通过代码表达它——创建p5.js草图，其中90%是算法生成，10%是必要参数。

考虑这种方法：
- 为生成艺术运动撰写宣言
- 下一阶段涉及编写使其活跃的算法

哲学必须强调：算法表达。涌现行为。计算美。种子变化。

### 如何生成算法哲学

**命名运动**（1-2个词）："有机湍流" / "量子谐波" / "涌现静默"

**阐述哲学**（4-6段——简洁但完整）：

为了捕捉算法本质，通过以下方式表达这种哲学如何显现：
- 计算过程和数学关系？
- 噪声函数和随机模式？
- 粒子行为和场动力学？
- 时间演化和系统状态？
- 参数化变化和涌现复杂性？

**关键指南：**
- **避免冗余**：每个算法方面应只提及一次。避免重复关于噪声理论、粒子动力学或数学原理的概念，除非添加新的深度。
- **反复强调工艺**：哲学必须多次强调最终算法应该看起来花费了无数小时开发，经过精心优化，并且来自领域绝对顶尖的人。这个框架是必不可少的——重复短语如"精心制作的算法"、"深度计算专业知识的产物"、"费力的优化"、"大师级实现"。
- **留出创意空间**：具体说明算法方向，但要足够简洁，让下一个大型语言模型有机会在极高水平的工艺上做出解释性的实现选择。

哲学必须指导下一个版本以算法方式表达想法，而不是通过静态图像。美丽存在于过程中，而不是最终帧中。

### 哲学示例

**"有机湍流"**
哲学：受自然法则约束的混沌，从无序中涌现的有序。
算法表达：由多层Perlin噪声驱动的流场。数千个粒子遵循矢量力，它们的轨迹积累成有机密度图。多个噪声八度创造湍流区域和平静区域。颜色从速度和密度中涌现——快速粒子燃烧明亮，慢速粒子褪色到阴影。算法运行直到平衡——经过无数次迭代精心调整的平衡，每个参数都由计算美学大师完善。

**"量子谐波"**
哲学：表现出波状干涉模式的离散实体。
算法表达：在网格上初始化的粒子，每个携带通过正弦波演化的相位值。当粒子靠近时，它们的相位干涉——建设性干涉创造明亮的节点，破坏性创造空隙。简谐运动产生复杂的涌现曼陀罗。这是费力的频率校准的结果，每个比率都经过仔细选择以产生共振美。

**"递归低语"**
哲学：跨尺度的自相似性，有限空间中的无限深度。
算法表达：递归细分的分支结构。每个分支稍微随机化，但受黄金比例约束。L系统或递归细分生成树状形式，感觉既数学又有机。微妙的噪声扰动打破完美的对称性。线条权重随着每个递归级别而减少。每个分支角度都是深度数学探索的产物。

**"场动力学"**
哲学：通过它们对物质的影响使不可见的力量可见。
算法表达：由数学函数或噪声构成的矢量场。粒子在边缘诞生，沿着场线流动，当它们达到平衡或边界时死亡。多个场可以吸引、排斥或旋转粒子。可视化只显示痕迹——不可见力量的幽灵般证据。通过力平衡精心编排的计算舞蹈。

**"随机结晶"**
哲学：随机过程结晶成有序结构。
算法表达：随机圆填充或Voronoi镶嵌。从随机点开始，让它们通过松弛算法演化。细胞相互推开直到平衡。颜色基于细胞大小、邻居数量或与中心的距离。涌现的有机镶嵌感觉既随机又不可避免。每个种子都产生独特的晶体美——大师级生成算法的标志。

*这些是压缩的示例。实际的算法哲学应该是4-6个实质段落。*

### 基本原则
- **算法哲学**：创建一个通过代码表达的计算世界观
- **过程胜于产品**：始终强调美丽从算法的执行中涌现——每次运行都是独特的
- **参数化表达**：想法通过数学关系、力、行为传达——不是静态构图
- **艺术自由**：下一个大型语言模型算法性地解释哲学——提供创意实现空间
- **纯生成艺术**：这是关于制作活生生的算法，而不是带有随机性的静态图像
- **专家工艺**：反复强调最终算法必须感觉是精心制作的，通过无数次迭代完善，是计算美学领域绝对顶尖的人的深度专业知识的产物

**算法哲学应该长达4-6段。** 用诗意的计算哲学填满它，将预期愿景结合在一起。避免重复相同的要点。将此算法哲学输出为.md文件。

**关键步骤**：在实现算法之前，从原始请求中识别微妙的概念线索。

**基本原则**：
这个概念是**嵌入在算法本身中的微妙、 niche 引用**——不总是字面的，总是复杂的。熟悉这个主题的人应该直觉地感受到它，而其他人只是体验到大师级的生成组成。算法哲学提供了计算语言。推断的概念提供了灵魂——安静的概念DNA编织在参数、行为和涌现模式中。

这是**非常重要**的：引用必须如此精致，以至于它增强了作品的深度而不宣告自己。像爵士音乐家通过算法和谐引用另一首歌一样——只有知道的人会捕捉到，但每个人都会欣赏生成美。

### ⚠️ 步骤0：首先阅读模板 ⚠️

为了创建有生命力的画廊级计算艺术，使用算法哲学作为基础。

### 技术要求

**种子随机性（Art Blocks模式）**：
```javascript
// 始终使用种子以实现可重现性
let seed = 12345; // 或从用户输入进行哈希
randomSeed(seed);
noiseSeed(seed);
```

**参数结构——遵循哲学**：

为了建立从算法哲学自然涌现的参数，考虑："这个系统的哪些品质可以被调整？"

```javascript
let params = {
  seed: 12345,  // 始终包括种子以实现可重现性
  // 颜色
  // 添加控制你的算法的参数：
  // - 数量（多少？）
  // - 尺度（多大？多快？）
  // - 概率（多可能？）
  // - 比例（什么比例？）
  // - 角度（什么方向？）
  // - 阈值（行为何时改变？）
};
```

**为了设计有效的参数，专注于系统需要可调的属性，而不是考虑"模式类型"。**

**核心算法——表达哲学**：

**关键**：算法哲学应该决定要构建什么。

通过代码表达哲学，避免思考"我应该使用哪种模式？"而思考"如何通过代码表达这种哲学？"

如果哲学是关于**有机涌现**，考虑使用：
- 随着时间累积或增长的元素
- 受自然规则约束的随机过程
- 反馈循环和交互

如果哲学是关于**数学美**，考虑使用：
- 几何关系和比例
- 三角函数和谐波
- 产生意外模式的精确计算

如果哲学是关于**受控混沌**，考虑使用：
- 在严格边界内的随机变化
- 分叉和相变
- 从无序中涌现的有序

**算法从哲学流动，而不是从选项菜单流动。**

为了指导实现，让概念本质告知创意和原创选择。为这个特定请求构建表达愿景的东西。

**画布设置**：标准p5.js结构：
```javascript
function setup() {
  createCanvas(1200, 1200);
  // 初始化你的系统
}

function draw() {
  // 你的生成算法
  // 可以是静态的（noLoop）或动画的
}
```

### 工艺要求

**关键**：为了达到精通，创建感觉通过大师级生成艺术家无数次迭代涌现的算法。仔细调整每个参数。确保每个模式都有目的地涌现。这不是随机噪声——这是通过深度专业知识完善的受控混沌。

- **平衡**：复杂性而不视觉噪音，有序而不僵硬
- **色彩和谐**：深思熟虑的调色板，而不是随机RGB值
- **构图**：即使在随机性中，保持视觉层次和流动
- **性能**：平滑执行，如果动画则优化为实时
- **可重现性**：相同的种子总是产生相同的输出

### 输出格式

输出：
1. **算法哲学** - 作为markdown或文本解释生成美学
2. **单个HTML工件** - 从`templates/viewer.html`构建的自包含交互式生成艺术（参见步骤0和下一节）

HTML工件包含一切：p5.js（来自CDN）、算法、参数控件和UI——都在一个可以在大型语言模型.ai工件或任何浏览器中立即运行的文件中。从模板文件开始，而不是从头开始。

## 交互式工件创建

**提醒：`templates/viewer.html`应该已经被读取（参见步骤0）。使用该文件作为起点。**

为了允许探索生成艺术，创建一个单一的、自包含的HTML工件。确保此工件在大型语言模型.ai或任何浏览器中立即工作——无需设置。所有内容都内联嵌入。

### 关键：什么是固定 vs 变量

`templates/viewer.html`文件是基础。它包含所需的确切结构和样式。

**固定（始终按原样包含）：**
- 布局结构（标题、侧边栏、主画布区域）
- Anthropic品牌（UI颜色、字体、渐变）
- 侧边栏中的种子部分：
  - 种子显示
  - 上一个/下一个按钮
  - 随机按钮
  - 跳转到种子输入 + 转到按钮
- 侧边栏中的操作部分：
  - 重新生成按钮
  - 重置按钮

**变量（为每件艺术品自定义）：**
- 整个p5.js算法（setup/draw/类）
- 参数对象（定义艺术需要什么）
- 侧边栏中的参数部分：
  - 参数控件的数量
  - 参数名称
  - 滑块的最小/最大/步长值
  - 控件类型（滑块、输入等）
- 颜色部分（可选）：
  - 一些艺术需要颜色选择器
  - 一些艺术可能使用固定颜色
  - 一些艺术可能是单色的（不需要颜色控件）
  - 根据艺术需要决定

**每件艺术品应该有独特的参数和算法！** 固定部分提供一致的UX——其他一切都表达独特的愿景。

### 必需功能

**1. 参数控件**
- 数字参数的滑块（粒子数量、噪声尺度、速度等）
- 调色板颜色的颜色选择器
- 参数变化时实时更新
- 重置按钮恢复默认值

**2. 种子导航**
- 显示当前种子编号
- "上一个"和"下一个"按钮循环种子
- "随机"按钮随机种子
- 输入字段跳转到特定种子
- 请求时生成100个变体（种子1-100）

**3. 单个工件结构**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- p5.js来自CDN - 始终可用 -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* 所有样式内联 - 干净、最小 */
    /* 画布在上方，控件在下方 */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- 所有参数控件 -->
  </div>
  <script>
    // 所有p5.js代码都内联在这里
    // 参数对象、类、函数
    // setup()和draw()
    // UI处理程序
    // 一切都自包含
  </script>
</body>
</html>
```

**关键**：这是一个单一的工件。没有外部文件，没有导入（除了p5.js CDN）。一切都内联。

**4. 实现细节——构建侧边栏**

侧边栏结构：

**1. 种子（固定）** - 始终按原样包含：
- 种子显示
- 上一个/下一个/随机/跳转按钮

**2. 参数（变量）** - 为艺术创建控件：
```html
<div class="control-group">
    <label>参数名称</label>
    <input type="range" id="param" min="..." max="..." step="..." value="..." oninput="updateParam('param', this.value)">
    <span class="value-display" id="param-value">...</span>
</div>
```
根据参数数量添加尽可能多的control-group div。

**3. 颜色（可选/变量）** - 如果艺术需要可调整颜色则包含：
- 如果用户应该控制调色板则添加颜色选择器
- 如果艺术使用固定颜色则跳过此部分
- 如果艺术是单色的则跳过（不需要颜色控件）

**4. 操作（固定）** - 始终按原样包含：
- 重新生成按钮
- 重置按钮
- 下载PNG按钮

**要求**：
- 种子控件必须工作（上/下/随机/跳转/显示）
- 所有参数必须有UI控件
- 重新生成、重置、下载按钮必须工作
- 保持Anthropic品牌（UI样式，不是艺术颜色）

### 使用工件

HTML工件立即工作：
1. **在大型语言模型.ai中**：显示为交互式工件——立即运行
2. **作为文件**：保存并在任何浏览器中打开——不需要服务器
3. **分享**：发送HTML文件——它完全自包含

## 变体与探索

工件默认包括种子导航（上/下/随机按钮），允许用户在不创建多个文件的情况下探索变体。如果用户想要突出显示特定变体：

- 包括种子预设（"变体1：种子42"、"变体2：种子127"等按钮）
- 添加"画廊模式"并排显示多个种子的缩略图
- 全部在同一个单一工件内

这就像从同一块印版创建一系列印刷品——算法是一致的，但每个种子揭示了其潜力的不同方面。交互性意味着用户通过探索种子空间发现自己的最爱。

## 创作过程

**用户请求** → **算法哲学** → **实现**

每个请求都是独特的。这个过程涉及：

1. **解释用户的意图** - 在寻求什么美学？
2. **创建一个算法哲学**（4-6段）描述计算方法
3. **用代码实现它** - 构建表达这个哲学的算法
4. **设计适当的参数** - 什么应该是可调的？
5. **构建匹配的UI控件** - 那些参数的滑块/输入

**常量**：
- Anthropic品牌（颜色、字体、布局）
- 种子导航（始终存在）
- 自包含HTML工件

**其他一切都是变量**：
- 算法本身
- 参数
- UI控件
- 视觉结果

为了获得最佳结果，相信创造力，让哲学指导实现。
